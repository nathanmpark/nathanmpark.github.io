What are the best practices associated with using classes vs. ids?

September 6, 2015

CSS is essential to adding style and detail to your existing html document. While basic elements like <h1>, <p>, or <div> can exist in your document, classes and ids are used for additional styling. The difference lies within the specificity of these two components

IDs are unique

  • Each element can have only one ID
  • Each page can have only one element with that ID

Classes are not unique

  • You can use the same class on multiple elements
  • You can use multiple classes on the same element

There are many ways in which you can implement these two component, but the main factor is dictated by repetition and where you might have to reuse a certain CSS styling. There are many different CSS conventions: OOCSS, Atomic, BEM, SMACSS. Ultimately the best practices are to code CSS classes and IDs so that they are structured enough to be re-used by other developers and such that they follow responsive design principles.
