Big O Complexity & Notation

Defining the Performance and Complexity of an Algorithm

October 6, 2015

Algorithms are an essential part to computer science and programming in general. They are used in sorting information, searching data, and processing system inputs. As systems scale, programming languages are put to the test in how effeciently they run. However, it is the job of the programmer to know how their code is being structured and if it will run into any performance issues in the long run. Big O notation effectively measures the the performance and complexity of an algorithm by comparing its worst case scenario. The Big O complexity can then be used to compare algorithms based on execution time and space used in memeory or on disk.

Big O notation is the relative representation of the complexity of an algorithm.

It is important when considering Big O notation to remember you can only compare algorithms that are similar in nature. You cannot compare an alogrithm that calculates multiplcation to an algorithm that sorts an array. However, a comparison of two similar algorithms can provide meaningful results for future design. (e.g. two search algorithms).
